# Brand System

Phase: 10 - Branding Run

Purpose:
- Translate the MyYado product model into enforceable brand and UI rules.
- Keep visual decisions tied to curation, trust, discovery, and Sharetribe constraints.
- Prevent old-repo dependency or generic template drift.

Inputs:
- `docs/product/domain-model.md`
- `docs/product/use-cases.md`
- `docs/product/sharetribe-mapping.md`
- `docs/product/transaction-and-search-model.md`
- Native Sharetribe flows: search, listing, checkout, provider listing creation

Core principle:
Define the system first. Apply it selectively. No UI change without a brand justification.

Brand files:
- `brand-brief.md`: promise, curation, cultural positioning, voice, anti-patterns
- `visual-direction.md`: visual rules, UX constraint, image system, theme tokens
- `page-priorities.md`: page roles, discovery model, supply constraint
- `trust-system.md`: trust signals and information clarity rules
- `market-scope.md`: launch markets and geographic curation rules
- `collections-model.md`: editorial collection system
- `template-vs-custom.md`: customization boundaries
- `branding-run-backlog.md`: prioritized UI work and front-end truth audit

Baseline statement:
MyYado is a curated marketplace for better stays, not more stays. The brand should feel calm,
specific, trustworthy, and ready to transact.
