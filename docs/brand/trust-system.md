# Trust System

Trust is communicated through:

- Structured listings with clear, consistent fields
- Provider identity through profile, story, and responsiveness
- Reviews that are prominent and legible
- Platform signals:
  - curated badge
  - featured status

## Structured Listings

Required trust fields:
- title
- description
- location
- images
- price
- availability
- Stay Type
- Sleeps
- bedrooms
- beds
- bathrooms
- amenities

Optional trust fields:
- bathing
- dining
- transfer availability
- languages
- private arrival instructions
- Provider internal notes

Rules:
- Public fields should help Travelers decide.
- Private fields should help Providers operate.
- Do not expose private arrival instructions publicly.

## Provider Identity

Provider trust should come from:
- profile name
- profile image
- Provider story
- response behavior
- reviews
- clear relationship to the Listing

Avoid anonymous or purely transactional listing cards when Provider data is available.

## Reviews

Reviews should:
- be visible near decision points
- remain tied to completed Transactions
- use Traveler/Provider vocabulary
- avoid decorative treatment that makes the content harder to scan

## Platform Signals

Curated badge:
- means the Listing passed the MyYado quality bar
- should appear only when the platform can stand behind it

Featured status:
- means the Listing is intentionally promoted
- should never replace clear price, availability, or review information

## Trust Breakers

Avoid:
- hidden information
- ambiguous pricing
- unclear availability
- unclear Provider identity
- collection labels that do not match actual tags
- "Reservation" language before Provider acceptance
